New in 0.3.4

Multiple Syntax for Hierarchical Tags

Hierarchical tags can now be defined using either parenthesis notation, or dot notation. The following waypoints are equivalent:

//@tag a(b(c))
//@tag a.b.c

Refactoring operations will scan the lines and keep your previous syntax.

Code Hints Sorted According to Relevance

Code completion proposals now check to see what the parent tag of what you are currently writing is. The children of this tag are listed first in the proposal.

TagSEA Registration

It is now much easier to participate in the TagSEA studies. You can register TagSEA and get a unique user id. Then, you can upload XML versions of the tags that you have created directly to a server at the University of Victoria. There is no more need to hassle with sending an email with the tags attached. You can also choose not to register, or to remain entirely anonymous when you register.

TagSEA Logging

You can choose to log your interactions with TagSEA in order to participate more fully in the TagSEA studies. If you have registered TagSEA, the logs of your interactions will be automatically uploaded to a server at the University of Victoria. You can view all of the logged information at any time using the TagSEA Log view.
