Your First Tag

TagSEA is currently coupled with the Eclipse Java Development Toolkit (JDT). There are plans to expand TagSEA to allow tags to be associated with arbitrary data, but for now the only way to define a tag is within the eclipse Java editor. To do this, simply open a Java file in the Java editor, and use a comment to define a tag:

//@tag mytag : This is a comment for my tag

Tags can appear within any valid Java comment. These are all equivalent:

//@tag mytag : This is a comment for my tag

/*
* @tag mytag : This is a comment for my tag
*/

/**
* @tag mytag : This is a comment for my tag
*/

The format is as follows:

@tag tagname <... more-tag-names> <: comment text > < [ meta-data ] >

Where bold text is required, and text inside <> is optional.

Technically, this line isn't actually for defining a tag, it defines what is called a waypoint. That is, it defines where a specific instance of a tag exists, as well as some information for that tag (the comment and the meta-data). However, if a tag with name tagname doesn't already exist, this comment will also define a new tag for tagname.

To make it easier to organize and visualize tags, a hierarchy of tags is supported. You can define the hierarchy within the tag name. This is done by placing child tags within parenthesis (()). For example, suppose you have several bugs in a program, and you would like to track them using tags. But, you also have two bug tracking mechanisms (say, sourceforge and bugzilla). You can tag the problem code using a hierachy:

//@tag bug(sourceforge(4353846))

The above will mark your code with the tag 4353846 inside the sourceforge list of bugs. For bug # 3485 in bugzilla, use:

//@tag bug(bugzilla(3485))

And, as of version 0.3.4, you can also use dot-notation for hierarchies. The following is equivalent to the above:

//@tag bug.bugzilla.3485

Note that it is possible to have multiple tags use the same comment and meta data for a waypoint. This tag is perfectly legal:

//@tag mytag mytagtwo : a comment for two tags

The comment comes after a colon (:) sign, and will be assigned to a waypoint for both mytag and mytagtwo.

If you want to add extra information about the waypoint, you can do so within meta data. Meta data is written by placing semi-colon (;) separated key/value pairs within square brackets. The meta data elements that are currently supported are the author and the date of the waypoint. Here is an example of a waypoint with meta data:

//@tag mytag : a comment [Author = me;Date = 14/07/06 11:42 AM;]

Again, the comment is entirely optional. The following is also legal:

//@tag mytag [Author = me;Date = 14/07/06 11:42 AM;]

Meta data can also be split accross multiple lines:

/*
 * @tag mytag [
 * Author = me;
 * Date = 14/07/06 11:42 AM;
 *]
 */

However, tag names and comments must all be on the first line of the tag/waypoint definition.

Experiment with writing tags, and see how useful they can be. TagSEA offers automatic code completion to make the process even smoother. Simply press <Ctrl-space> after entering the @tag keyword to see the available hints.
